# Git Command Performance Issues

## Problem
Git commands sometimes appear to hang or take a long time, especially `git commit`.

## Root Causes
1. **No active hooks** - Only sample hooks exist, so hooks aren't the issue
2. **PowerShell delays** - PowerShell may have delays returning control
3. **Git processing** - Git may be doing background work (indexing, etc.)

## Solutions

### Fast Commit (Skip Hooks)
```powershell
git commit --no-verify -m "message"
```

### Batch Operations
Instead of multiple separate commands, use:
```powershell
git add . ; git commit --no-verify -m "message"
```

### Check Git Status First
Always check status before committing to avoid surprises:
```powershell
git status --short
```

## Best Practices
- Use `--no-verify` for faster commits during development
- Batch git operations when possible
- Check status before committing
- Use `git add -A` instead of `git add .` for consistency
